## 📘 Common Table Expressions (CTEs)

A **CTE** is a temporary result set that exists only during query execution.

**Benefits**
- Improves readability  
- Breaks large queries into smaller logical steps  
- Makes complex transformations easier to understand  
- Useful when aggregated results need further analysis  
- Supports recursive queries for hierarchies  

**Syntax**
```sql
WITH cte_name AS (
    SELECT ...
)
SELECT *
FROM cte_name;
```

## Example 1: Average Release Year by Content Type
### Query

```sql
WITH avg_release AS (
    SELECT
        ROUND(AVG(release_year),0) AS avg_releaseyr,
        show_type
    FROM netflix
    GROUP BY show_type
)
SELECT *
FROM avg_release;
```
## Explanation

- The **CTE** calculates the average release year for each content type (Movie, TV Show).  
- The result is stored temporarily in the CTE named `avg_release`.  
- The main query then retrieves and displays the calculated values from this temporary dataset.  

## Example 2: Multiple CTEs

### Problem  
Count total Movies and TV Shows separately.

### Query
```sql
WITH total_movie AS (
    SELECT COUNT(*) AS movie_count
    FROM netflix
    WHERE show_type = 'Movie'
),
total_tvshow AS (
    SELECT COUNT(*) AS tvshow_count
    FROM netflix
    WHERE show_type = 'TV Show'
)
SELECT *
FROM total_movie, total_tvshow;
```
## Explanation

- Two independent **CTEs** are created:  
  - `total_movie` → counts all movies.  
  - `total_tvshow` → counts all TV shows.  

- The final query selects both results together in one combined output.

## 📦 Subqueries

A **subquery** is a query nested inside another query.  
They are useful when a query needs a value generated by another query.

---

### Example 1: Movies Released After Average Release Year

#### Query
```sql
SELECT
    title,
    release_year
FROM netflix
WHERE release_year >
(
    SELECT AVG(release_year)
    FROM netflix
)
ORDER BY release_year DESC;
```
## Explanation

- **Inner query** → Calculates the average release year across all titles.  
- **Outer query** → Retrieves only the content released after that average year.  

---

## Flow
1. **Calculate Average Release Year**  
2. **Compare Each Row** against the average  
3. **Return Matching Content** (titles released after the average year)

### Example 2: Latest Released Content

#### Query
```sql
SELECT
    title,
    release_year
FROM netflix
WHERE release_year =
(
    SELECT MAX(release_year)
    FROM netflix
);
```
## Explanation

- Inner query finds the latest release year.
- Outer query returns all content released in that year.

## 🏆 Interview-Style CTE Example

### Business Question    
Which movies/shows did each user like more than they usually do?

### Query
```sql
WITH user_avg AS (
    SELECT
        user_id,
        ROUND(AVG(rating),0) AS avg_rating
    FROM ratings
    GROUP BY user_id
)
SELECT
    u.user_name,
    n.title,
    r.rating,
    ua.avg_rating
FROM ratings r
JOIN user_avg ua
    ON r.user_id = ua.user_id
JOIN users u
    ON r.user_id = u.user_id
JOIN netflix n
    ON r.show_id = n.show_id
WHERE r.rating > ua.avg_rating;
```
## Explanation

- Step 1 → Calculate each user’s average rating.
- Step 2 → Join averages back to the ratings table
- Step 3 → Return only ratings higher than the user’s average.

## ⚖️ When to Use GROUP BY + HAVING
Use when you need aggregated results and want to filter those groups.

### Examples

- Departments with average salary above 60,000
- Genres with more than 100 movies
- Users with average ratings above 4

### Pattern
```sql
GROUP BY ...
HAVING ...
```

## 📊 When to Use a Subquery
Use when you need a single value or result from one query inside another.

### Examples

- Movies released after the average release year
- Content released in the latest year
- Users above the overall average rating

### Pattern
```sql
SELECT ...
WHERE column >
(
    SELECT ...
)
```
## 📘 When to Use a CTE
Use when the query contains multiple logical steps or an intermediate result needs to be reused.

### Examples

- Calculate user averages then compare ratings
- Aggregate data then join it to other tables
- Multi-step analytics transformations

### Pattern
```sql
WITH step1 AS (...),
step2 AS (...)
SELECT ...
FROM step2;
```

## 📈 When to Use Window Functions
Use when the query contains multiple logical steps or an intermediate result needs to be reused.

### Examples

- Department average salary beside every employee
- Ranking content by rating
- Running totals
- Moving averages

### Pattern
```sql
AVG(...) OVER(...)
RANK() OVER(...)
ROW_NUMBER() OVER(...)
```

## 🌳 Recursive CTEs
A **Recursive CTE** references itself and is used for hierarchical data such as:
- Employee → Manager relationships
- Organization charts
- Folder structures
- Parent-child categories

## Complete Query
  ```sql
WITH cteReports (EmpID, FirstName, MgrID, EmpLevel) AS (
    -- Anchor Member
    SELECT
        EmployeeID,
        FirstName,
        ManagerID,
        1 AS EmpLevel
    FROM Employees
    WHERE ManagerID IS NULL

    UNION ALL

    -- Recursive Member
    SELECT
        e.EmployeeID,
        e.FirstName,
        e.ManagerID,
        r.EmpLevel + 1
    FROM Employees e
    JOIN cteReports r
    ON e.ManagerID = r.EmpID
)
SELECT *
FROM cteReports;
```
## Example Hierarchy
```
Michael
├── Emily
│   ├── Sophia
│   └── Ethan
└── Daniel
```
### Process

- Anchor → Find top-level managers.
- Recursive → Find employees reporting to them.
- Repeat until no rows remain.

## 📌 Day 4 Takeaways

- **[CTEs](ca://s?q=Explain_SQL_CTEs)**  
  Break complex queries into smaller logical steps, improve readability, and allow reuse of intermediate results.  

- **[Multiple CTEs](ca://s?q=Explain_multiple_CTEs_in_SQL)**  
  Useful when combining independent calculations (e.g., counting Movies vs TV Shows).  

- **[Subqueries](ca://s?q=Explain_SQL_subqueries)**  
  Best for one-time calculations inside another query, such as comparing against `AVG()` or `MAX()`.  

- **[GROUP BY + HAVING](ca://s?q=Explain_SQL_GROUP_BY_vs_HAVING)**  
  Filter aggregated results after grouping. Essential when conditions apply to aggregates rather than raw rows.  

- **[Window Functions](ca://s?q=Explain_SQL_window_functions)**  
  Perform calculations across groups without collapsing rows. Ideal for ranking, running totals, and moving averages.  

- **[Recursive CTEs](ca://s?q=Explain_recursive_CTEs)**  
  Handle hierarchical data like employee-manager structures or folder trees.  

---

### 🔑 Key Insights
- **CTE** → Multi-step SQL transformations.  
- **Subquery** → Embed one query’s result inside another.  
- **GROUP BY + HAVING** → Aggregate and filter groups.  
- **Window Function** → Keep rows while adding calculations.  
- **Recursive CTE** → Traverse hierarchies.
